CSV Import Data

Importing CSV Data into frames - CSV record types
CSV column/field names are all lower case, and when possible convert (with case changes) directly to the field name on the babylon/webGL asset.

Common CSV Columns (asset field)
title (name)
materialname (materialName)
color (diffuseColorName, ...)
texturepath, scaleu, scalev (diffuseTextureName, ...)
diffuse, ambient, emissive ('x' applies)
bmppath (bumpTextureName)
x (positionX)
y (positionY)
z (positionZ)
rx (rotationX)
ry (rotationY)
rz (rotationZ)
sx (scalingX)
sy (scalingY)
sz (scalingZ)

CSV Generation blockFlag values
  (block::blockFlag) = "scene" - root block for the generated scene and assets/links
  (block::blockFlag) = "basket" - selected item basket block
  (block::blockFlag) = "displaypositions" - preset list of positions for items; genericblockdata = "x1||y1||z1||x2||y2..."
  (block::blockFlag) = "displayfieldedits" - override fields in the layout/scene template; genericblockdata = "displaytitle||$template|| $assetName||$assetType|| $fieldname1:$fieldtype1|| $fieldname2:$fieldtype2..."
    $template - layout/product/asset; $assetName - [name]; $assetType - [asset]; $fieldtype - [csv column name]; $fieldtype - display type (image,num)
  (block::blockFlag) = "displayblock" - tags block for matching to sku/itemid based items

CSV Import Asset Types
meshtexture - a row for a mesh asset and optional texture asset (url)
  [url]: mesh path
block - row for a block asset
  basic fields - [materialname], [height], [width], [depth]
  [blockcode] (blockCode) - lookup field
  [blockflag] (blockFlag) - field flagging block as special case
  [genericblockdata] (genericblockdata) - import data (i.e. hashed position list)
blockchild - [parent] relates to title of parent block, parent must exist
  [materialname] = 'inherit' sets (inheritMaterial) = true
  [childtype](childType) - shape, mesh, camera, light, block
  [childname](childName) - title/name of Linked Asset - linked asset can be created later
textplane - a row for texture and shape (plane) assets showing fitted/sized 2D text
  [width, height, depth] - determine size
  [texturetext, texturetext2] - line 1 and 2 of text
  [textfontfamily, textfontcolor, textfontsize, textfontweight] - html/css font fields
  [texturetextrendersize] - 512 by default - the width of the container the text is rendered in
shape - shape asset with texture and color information (text info is for 3d text)
  [height](boxHeight), [width](boxWidth), [depth](boxDepth) - shape dimensions
  [shapetype] - box, cylindar, sphere, text (3d), plane, torus
  [textfontfamily, texttext, textdepth, textsize] - 3d text fields - fontfamily is css/html based
blockchildframe - adds a frame to a linked block (block child)
  [parent] - links to pre-existing block via title/name
  [childtype, name] - all linked/child blocks matching childType and childName (block/shape/mesh title)
  [...fields] - x, y, z, rx, ry, rz, sx, sy, sz, visibility, frameorder, frametime
message - adds message to "scene" - with index for camera stop
  - [index] determines the order the message appears in the animation
  - [asset] = "textplane" csv columns apply
image - adds image to "scene" - with index for camera stop
  - doesn't exist yet (to do)
product - adds product to "scene" - with index for camera stop
displaycamera - adds FollowCamera and follow block to "scene"
  [introtime, runlength, finishdelay, cameramovetime] - important fields for animation timing
  [startx, starty, startz, startrx, startry, startrz] - determine starting position for followed block
  [x, y, z, rx, ry, rz, cameraheightoffset, cameraradius] - starting position for follow camera
animationfinalize - final csv import to generate animation frames based on product/message/image/displaycamera data
  generates the follow block and frames for camera
  places displayblocks, etc in the basket for the items able to be selected
  frames to show and hide signs for "index" (message/image/product) items
